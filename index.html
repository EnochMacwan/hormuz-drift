Setting up WebGNOME with Hormuz data

1
Open WebGNOME Manual Setup

Click Open WebGNOME ↗ above. In WebGNOME, choose Manual Setup from the home screen.

2
Add your CMEMS currents file

In WebGNOME: Setup → Movers → Add a Mover → Grid Current. Upload the file:
cmems_mod_glo_phy_anfc_merged-uv_PT1H-i_...nc
(found in your project root folder). WebGNOME reads CF-compliant NetCDF natively.

3
Add wind forcing

Go to Movers → Add a Mover → Wind. Either enter a constant wind or use GOODS to fetch GFS wind for the Hormuz region (lat 23–30 N, lon 53–60 E).

4
Place the spill

Go to Spills → Add a Spill → Instantaneous. Set the lat/lon to match your release point from this model. Set the start time to match the data window.

5
Select oil type from ADIOS

For oil scenarios: Spills → select your spill → Oil → Browse ADIOS database. Search for the oil type matching your scenario (e.g. Arabian Medium for Hormuz crude).

6
Run and compare

Click Run Model. WebGNOME outputs an oil budget chart, trajectory animation, and beached mass — use these to validate and extend your Hormuz Drift results.

💡 Tip: For scripted / repeatable runs, download the PyGNOME script button above and run it locally with python pygnome_hormuz.py after pip install gnome.
