Release and physics

Waiting for data
Release point The map click where particles, people, or oil are introduced into the model.
Jitter Initial scatter radius around the release point. Higher values represent less certainty about the exact start location.
Diffusion K Random-walk mixing strength in m²/s. Larger K spreads the ensemble faster to mimic unresolved turbulence.
Duration How many hours the scenario runs forward from the selected release time.
Ensemble size Number of simulated particles. More particles produce smoother probability patterns but run slower.
Wind drift Adds wind-driven leeway and Stokes-style surface drift when the loaded dataset includes wind vectors.

Model honesty

Surface onlyNo subsurface shear, vertical mixing, or wave-spectrum resolved Stokes drift.
Hourly browser gridRTOFS fallback is packed as hourly output; source snapshots may be interpolated from coarser forecast files.
Shoreline riskStranding uses the model land mask, not high-resolution shoreline polygons or port infrastructure.
Operational handoffUse WebGNOME/OpenDrift for certified response planning, oil chemistry, and official products.

Response options

Mechanical recovery
Typical field efficiency 10–40 %. Reduced on high-viscosity emulsions.
In-situ burning
ISB burn rate ≈ 3.5 mm/min. Less effective on emulsions (water fraction > 25 %).
Chemical dispersant
Corexit-class. Most effective on light to medium crudes (API 20–35). Ineffective on HFO.

Data provenance

Currents come from the active dataset metadata: CMEMS when authenticated, or NOAA/NCEP RTOFS when the no-login real-time fallback is used. Wind vectors are embedded when the dataset refresh includes Open-Meteo/GFS or NCEP GFS forcing. Tiles by OpenStreetMap and CARTO.

Setting up WebGNOME with Hormuz data

1
Open WebGNOME manual setup

Click Open WebGNOME, then choose manual setup in the NOAA interface.

2
Add project currents

Add a grid-current mover and upload the CMEMS or NOAA RTOFS NetCDF currents file used for this project.

cmems_*.nc or rtofs_glo_2ds_*.nc
3
Add wind forcing

Add a wind mover using GOODS/GFS, a constant wind, or a prepared local wind file for the same time window.

4
Place the spill or release

Use the release latitude/longitude from this model, align the start time with the data window, then set duration and output options.

5
Run and compare

Compare WebGNOME trajectories, oil budget, and beached mass outputs against the browser animation.

Repeatable workflow: download the PyGNOME script and run it locally after installing NOAA GNOME dependencies.
